# Reporting Improvement Plan

## Objective

Improve the telemetry reporting pipeline so the final output is actionable, structured, and easier to consume.

## Current problems

- `reports/anomaly_collector.py` stores raw issue strings only.
- `reports/report_generator.py` groups anomalies by exact string match and emits repetitive cards.
- `reports/report_merger.py` writes a plain bullet list and does not add value.
- `main.py` generates only a single report card file; AI summary integration is not fully wired.
- Anomalies lack metadata such as source, severity, raw log evidence, and event context.

## Goals

1. Emit structured anomaly records from agents.
2. Persist anomalies with metadata in `reports/raw_anomalies.json`.
3. Generate a richer markdown report with:
   - executive summary
   - severity distribution
   - source distribution
   - top anomaly types
   - sample log entries
   - suggested fixes
4. Produce a final report file suitable for stakeholders.
5. Keep the reporting path extensible for AI summary and future dashboards.

## Action items

### 1. Normalize anomaly model

- Update `reports/anomaly_collector.py`:
  - accept both raw strings and structured anomaly objects.
  - normalize into records with fields:
    - `id`
    - `title`
    - `description`
    - `severity`
    - `source`
    - `category`
    - `detected_at`
    - `metadata.raw_line`
    - `metadata.event_type`
- Ensure each `collect(...)` call writes a consistent JSON array.

### 2. Standardize agent anomaly emission

- Update agents to pass structured anomaly objects or enriched metadata:
  - `agents/gps_agent.py`
  - `agents/battery_agent.py`
  - `agents/can_agent.py`
  - `agents/mqtt_agent.py`
  - `agents/net_agent.py`
- Include `source` and `event` metadata where available.
- Prefer anomaly IDs for grouping.

### 3. Improve report generation

- Update `reports/report_generator.py`:
  - read structured anomaly objects.
  - produce `report_cards.md` with sections:
    - summary statistics
    - severity distribution
    - source distribution
    - anomaly cards with examples
  - add `generate_final_report()` for `final_report.md`.
- Retire or replace `report_merger.py` with a true final-report generator if needed.

### 4. Integrate summary output

- Wire `main.py` to create both report card and final report files.
- Optionally call `llm/summary_agent.py` after reports are generated.
- Ensure AI summary can operate on structured anomalies if credentials exist.

### 5. Validate and document

- Run `python main.py` to verify report generation.
- Inspect generated files:
  - `reports/raw_anomalies.json`
  - `reports/report_cards.md`
  - `reports/final_report.md`
  - `reports/ai_summary.md`
- Add a small README note or code comment explaining the report schema and output files.

## Deliverables

- `reports/anomaly_collector.py` updated
- `reports/report_generator.py` updated
- `reports/final_report.md` generated by the improved pipeline
- `main.py` updated to run the new reporting path
- optional `reports/ai_summary.md` when AI key is configured

## Timeline

1. Normalize anomaly format and collector behavior.
2. Upgrade report generator output structure.
3. Update `main.py` and verify end-to-end run.
4. Review sample output and adjust card layout.
5. Optionally add AI summary and documentation.
