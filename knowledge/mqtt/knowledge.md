# MQTT Domain Knowledge

## Overview
The MQTT domain captures telemetry for message broker status, latency, and message throughput. Use this knowledge base to interpret MQTT connectivity health and message flow anomalies.

## Key MQTT Signals
- `MQTT_COUNT`: Total MQTT messages processed
- `MQTT_STATUS`: Broker connection state
- `MQTT_LATENCY`: Time delay between publish and receive

## Use Cases
- Detect broker disconnections and reconnection loops
- Identify latency spikes in message delivery
- Monitor broker availability and performance
